# **msit** #

## Brief Introduction ##

As a unified Ascend inference tool, MindStudio Inference Tools (msit) provides the integrated development function, helping users migrate models and debug and optimize performance and precision. Currently, the tool includes the benchmark, debug, analyze, convert, profile, llm, and tensor-view components.

**Attention.**

LLM component in the msit. The debug component provides the following functions:[msprobe](https://gitcode.com/Ascend/msprobe)	The corresponding optimization has been implemented in, and the msit functions are no longer evolved. You are advised to use the msprobe tool.

## Precautions ##

 *  All paths read and written by the tool. Only uppercase letters, lowercase letters, digits, underscores (_), slashes (/), dots (.), and hyphens (-) are allowed.
 *  To ensure security and minimize permissions, you are advised to install the msit tool as a common user. Do not use the msit tool as a high-privilege account such as root.
 *  Before using the msit tool, ensure that the umask value of the execution user is greater than or equal to 0027. Otherwise, the permission on precision data files and directories generated by the msit tool may be too high.
 *  Users must ensure that the least permission principle is used. For example, the other user cannot write the file input to the tool. In some function scenarios that have strict security requirements, the group user must not write the file input to the tool.
 *  When using the msit command line, check whether the current environment has an available and unique Python environment.

## Command Format ##

The msit tool is started in command line mode.

```
msit <TASK> <SUB_TASK> [OPT] [ARGS]
```

| Parameters   | Description                                                                                                                                                                                                                                                                        |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TASK         | Task type. Currently, the value can be debug, benchmark, analyze, convert, profile, llm, graph, or tensor-view. For details about the task, see.[Function Description of Each Component](#各组件功能介绍)	. You can also perform the`msit -h`command to view the list of supported tasks. |
| SUB_TASK     | `<TASK>`Subtask types contained under, starting with`debug`For example, you can run the following command:`msit debug -h`m command to view the list of sub-functions supported by each task.                                                                                       |
| OPT and ARGS | Options and parameters. The options and parameters under each task may be different.`debug`tasked`compare`For example, a subtask can be executed by`msit debug compare -h`command to obtain the options and parameters.                                                            |

For example, to enable the dump function of the LLM, run the following command:

```
msit llm dump <options>
```

## Function of each component ##

### llm ###

Offered[**MindIE**](https://www.hiascend.com/software/mindie)	And to the[**torchair**](./docs/glossary/README.md)	framed[Large Model Inference Debugging Tool](./docs/llm/README.md)	. For details about the modules, see the following table.

| Module   | Description                                                                                                                                                                                                                                                             |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| dump     | Provides the data dump function during large model inference, including ATB and PyTorch scenarios. For details about the usage guide, see.[ATB dump User Guide](./docs/llm/工具-DUMP加速库数据使用说明.md)	And to the[Pytorch dump User Guide](./docs/llm/工具-Pytorch场景数据dump.md)	. |
| compare  | Provides the automatic comparison function for large-scale model inference to quickly locate operator precision problems. For details, see.[Compare User Guide](./docs/llm/工具-大模型精度比对.md)	.                                                                             |
| opcheck  | Provides the single operator precision pre-check function of the acceleration library (ATB) to check whether the operator precision in the acceleration library meets the requirements. For details, see.[Opcheck User Guide](./docs/llm/工具-精度预检使用说明.md)	.              |
| errcheck | Provides the exception detection capability. Currently, only operator overflow detection is supported. For details, see.[Anomaly Detection Instructions](./docs/llm/工具-异常检测使用说明.md)	.                                                                                   |

### debug ###

Provides the one-stop debugging function to locate problems during user inference in traditional small models and ensure model correctness. The following table lists the included modules.

| Module  | Description                                                                                                                                                                                                                                                                                                                                                                         |
| ------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| dump    | Provides the data dump function in traditional small model scenarios and applies to TensorFlow, TensorFlow 2.0, ONNX, Caffe, and MindIE-Torch frameworks. For details, see.[Dump User Guide](./docs/debug/dump/README.md)	.                                                                                                                                                         |
| compare | Provides the automatic comparison function in traditional small model inference scenarios to locate problem operators. This function is applicable to TensorFlow, TensorFlow 2.0, ONNX, Caffe, and MindIE-Torch frameworks. For details, see.[Compare User Guide](./docs/debug/compare/README.md)	And to the[Converged pass comparison](./docs/debug/compare/融合算子匹配对应pass使用说明.md)	. |
| opcheck | Provides the precision pre-check function in traditional small model scenarios. It pre-checks the operator precision of dumped data and flushed data after GE inference and checks whether the kernel-level operator precision meets the requirements. For details, see.[Opcheck User Guide](./docs/debug/opcheck/README.md)	Currently, only TensorFlow 2.6.5 is supported.         |
| surgeon | Enables ONNX model optimization on the Ascend chip and provides the ONNX-based image modification function. For details, see.[Surgeon User Guide](./docs/debug/surgeon/README.md)	.                                                                                                                                                                                                 |

### analyze ###

Analyzes the support for migrating models from other platforms to the Ascend platform, including whether operators are supported, whether operator definitions meet constraints, and whether operator input is empty. For details, see.[Analyze User Guide](./components/analyze/README.md)	.

### convert ###

Converts model files of frameworks such as ONNX, TensorFlow, Caffe, and MindSpore into OM files and supports optimization. For details, see.[Convert User Guide](./docs/convert/README.md)	.

### profile ###

Provides the performance analysis function and is oriented to OM files. (Offline model converted from files such as ONNX) Analyze the performance of model inference on the Ascend and provide detailed performance data and related information about the entire network. For details, see.[Profile User Guide](./docs/profile/README.md)	.

### benchmark ###

Runs the inference program based on the specified inference model and tests the performance (including throughput and latency) of the inference model, helping users evaluate the performance of the inference model. For details, see.[Benchmark User Guide](./docs/benchmark/README.md)	.

### graph ###

Provides graph statistics, compression, capture, and performance analysis functions based on the Graph Engine (GE). For details, see.[Graph User Guide](./docs/graph/README.md)	.

### tensorview ###

Provides APIs for viewing tensors and performing chain slicing and transposing operations on tensors. For details, see.[Tensor-view User Guide](./docs/tensor_view/README.md)	.

### elb ###

Provides the Deepseek model in static and dynamic scenarios, and provides the load balancing affinity expert optimization policy. For details, see.[Load Balancing Algorithm User Guide](./docs/expert_load_balancing/工具-负载均衡亲和专家寻优.md)	.
