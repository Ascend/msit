# Guide to Using the msit Profile Analyze Function

## Brief Introduction

 * This tool is a performance optimization analysis tool for recommended scenarios. Currently, only the GE automatic convergence operator can be analyzed.

## How to Use

### Prerequisite

 * When the tool is running, it invokes the ATC tool to convert GE dump images. Before using the tool, ensure that the environment variables of CANN are sourced, for example, source /usr/local/Ascend/ascend-toolkit/set_env.sh.

### Using the entrance

After the msit-profile tool is installed, you can use the tool through the command line.

```bash
msit profile analyze --origin /tmp/op_summary_origin.csv --fused /tmp/op_summary.csv --ops-graph /tmp/ge_proto_00000001_graph_1_Build.txt
```

### Parameter Description

| Parameter name    | Description                                                                                                                                                                                                                                          | Mandatory. |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------- |
| --mode            | Configure the mode inference scenario, for example, single operator or graph mode. Currently, only graph mode is supported.                                                                                                                          | No.        |
| -f, --framework   | Configure the AI framework for model inference. Currently, only TensorFlow is supported.                                                                                                                                                             | No.        |
| --origin          | op_summary file collected when operator fusion is disabled.                                                                                                                                                                                          | Yes        |
| --fused           | Op_summary file collected after operator fusion is enabled.                                                                                                                                                                                          | Yes        |
| -ops, --ops-graph | Graph generated by the GE, for example, ge_proto_xx_Build.txt. If there are multiple diagram files, you can enter the parent directory, for example, --ops-graph /tmp. Do not place them in the same directory as the diagram files of other models. | Yes        |
| -o, --output      | Indicates the path for saving performance analysis results. The default path is the current path.                                                                                                                                                    | No.        |
| -h, --help        | Help information about the tool.                                                                                                                                                                                                                     | No.        |

## Output Introduction

> Performance Analysis Results

After the execution is complete, a profile_analysis.csv file is generated in the specified output path. The file records the performance analysis results of the fusion operator. Currently, only the two types of fusion operators related to automatic fusion are supported. The operator types are AscBackend and FusedAscBackend.

 * HBM refers to the amount of data to be transferred, in KB. The analysis result has many columns. The meaning of each column is described as follows:

| Column Name                 | Description                                                                                                                                                     |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Fuse OpName                 | Indicates the operator name of the fusion operator.                                                                                                             |
| Fuse OpType                 | Operator type of the fusion operator.                                                                                                                           |
| Origin Ops                  | Original operator 2-tuple list corresponding to the fusion operator, recording the operator name and operator type of each original operator.                   |
| Fused Durations(us)         | Total time required by the fusion operator.                                                                                                                     |
| Origin Durations(us)        | Total duration of all single operators before convergence                                                                                                       |
| Time Ratio                  | Operator time consumed after fusion divided by the total operator time consumed before fusion                                                                   |
| Time Difference             | Operator time required after fusion minus operator time required before fusion                                                                                  |
| HBMs Difference             | HBM after fusion minus the sum of HBM before fusion                                                                                                             |
| HBMs Ratio                  | After the fusion operator HBM divided by the sum of the operators HBM before the fusion operator                                                                |
| Fused HBMs(KB)              | Input and output HBMs of the operator after fusion (unit: KB)                                                                                                   |
| Origin Duration(us) Each Op | Indicates the time consumption and list of each original operator. The operator name and corresponding time consumption of each original operator are recorded. |
| Origin HBMs Each Op(KB)     | HBM list of each original operator, which records the input HBM and output HBM of each original operator.                                                       |
| Origin HBMs Total(KB)       | Sum of input and output HBMs of all original operators                                                                                                          |
| Not Found Origin Op         | Name of the operator whose profiling information is not collected in the operator before fusion.                                                                |

> ATC Conversion File

During the running, the ATC tool is invoked to convert the GE dump file. The converted file is saved in the path specified by the --output parameter. The file name is the same as that before the conversion, and the extension name is json, that is, ge_proto_00000001_graph_1_Build.txt -> ge_proto_00000001_graph_1_Build.json.
