# User Guide to the Msit Benchmark Function

## Brief Introduction

This document describes the benchmark function, which is used to run the inference program based on the specified inference model and test the performance (including the throughput and delay) of the inference model.

## Tool installation

 * For details about how to install the tool, see.[MSIT Integrated Tool User Guide](../install/README.md)    
    If you are in`msit install benchmark`An error is reported and the message is displayed as required.`use --no-check-certificate`, available through`msit install benchmark --no-check`Install.
    Note that the --no-check-certificate parameter, that is, the --no-check-certificate parameter, skips checking the certificate information of the target website, which poses security risks. Therefore, you need to use the parameter with caution and bear the consequences.

## How to Use

### Function Description

#### Using the entrance

The benchmark inference function can be used to start the model test through the msit command line. The startup mode is as follows:

```bash
msit benchmark --om-model *.om
```

In the preceding command, \* indicates the name of the OM offline model file converted by the ATC tool.

#### Security risk warning

Before uploading a model file to the tool, ensure that the file is secure and reliable. If the source of the model file provides a verification value such as SHA256, you must verify the file to ensure that the model file is not tampered with.

#### Parameter Description

The benchmark inference function can be used to cope with various test scenarios and implement other auxiliary functions by configuring different parameters.

Parameters are classified into basic function parameters and advanced function parameters by function type.

 * **Basic function parameters: include the input file and format, debug, inference times, warm-up times, specified running device, and help information.**
 * **Advanced function parameters: include benchmark inference test parameters in dynamic ranking and dynamic shape scenarios, and profiler or dump data acquisition.**

**Note: In the following parameters, the parameter and value can be separated by a space or an equal sign (=). For example, --debug 1 or --debug=1.**
