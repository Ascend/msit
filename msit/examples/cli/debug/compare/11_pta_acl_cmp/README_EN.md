# Set_label Code Insertion Instructions #

The precision comparison in the large model acceleration library uses the data on the PyTorch Ascend (pta) side as the benchmark data to compare the inference data in the acceleration library (acl) with the pta data, helping developers locate problems in the acceleration library.

## 1. Comparison level ##

Operations in the acceleration library are divided into three granularities: Op, Layer, and Model. During the development of the acceleration library, the code replacement on the PTA side is also divided into the three granularities.

## 1.1 Replacing an Op ##

For the replacement at the Op granularity, the PTA can obtain the input and output data of an operation. An operation has multiple kernels, and the PTA cannot obtain the data of the kernels in the operation. Therefore, the acceleration library needs to provide data.


If the precision of the input/output data of an operation is compared, it is called high-level. If the precision of the input/output data of the operation kernel is compared, it is called low-level.

## 1.2 Layer Replacement ##

For layer-based replacement, the PTA can obtain the input/output data of the entire layer. A layer has multiple operations or kernels, and the PTA cannot obtain the data of internal operations or kernels. Therefore, the acceleration library needs to provide data.

If the precision of the input/output data of a layer is compared, the precision is called high-level. If the precision of the input/output data of the kernel inside the layer is compared, the precision is called low-level.

## 1.3 Replacing Models ##

For model-based replacement, the PTA can obtain the input and output data of the entire model. A model has multiple layers or operations. The PTA cannot obtain the data of internal layers or operations. Therefore, the acceleration library needs to provide data.

If the precision of the input/output data of the model is compared, it is called high-level. If the precision of the input/output data of the internal layer or operation of the model is compared, the precision is called low-level.

## 2. Interface Description ##

## 2.1 API Introduction ##

### 2.1 set_label ###

set_label(data_src, data_id, data_val, tensor_path)

Description: This API is used to add labels to the code on the PTA side of a model and record the source, ID, value, and path of the data to be compared.

Return value: None.

| Parameter name | Meaning                                      | Mandatory or not | Instructions for use                                                                                                                                                                                                                    |
| -------------- | -------------------------------------------- | ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| data_src       | Data source                                  | Yes              | Data type: str. The value can be acl or pta. acl indicates the data of the acceleration library, and pta indicates the data of the PyTorch Ascend, that is, the benchmark data.                                                         |
| data_id        | ID of the data.                              | Yes              | Data type: str, which is generated by the gen_id() interface. Data with the same ID indicates the data compared in pairs.                                                                                                               |
| data_val       | Value of the data.                           | No.              | Data type: torch.Tensor. When data_src is set to pta, this parameter is mandatory. When data_src is set to acl, this parameter is mandatory for high-level comparison. For low-level comparison, you do not need to set this parameter. |
| tensor_path    | Data dump path in the acceleration database. | No.              | Data type: str. When data_src is an ACL, during low-level comparison, you need to provide the dump operation or kernel data path in the acceleration library.                                                                           |

### 2.2 gen_id ###

gen_id()

Function: This API is used to generate data IDs based on timestamps.

Return value: data_id. The value is of the character string type.

### 2.3 set_task_id ###

set_task_id()

Description: This API is used to set the data directory for dumping on the acceleration library. It is recommended that this API be invoked before each round of dialog to compare the precision of multiple rounds of dialogs.

Return value: None.

## 2.2. Command Line Description ##

Use the format:

```
msit debug compare aclcmp xx_args
```

The optional parameters are as follows:

| Parameter name | Meaning                                                                                                                                            |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| --exec         | Run the following command to start the large model inference script: You are advised to use bash xx.sh args or python3 xx.py to start the process. |

# 3. Example #

Install the msit tool before using it. For details about the installation guide, see the following:https://gitcode.com/Ascend/msit/blob/master/msit/docs/install/README.mdThis section uses chatglm-6b as an example to describe how to use the acceleration library precision comparison tool.

1.  Set task_id.
    
    Set task_id and modify main_performance.py before each round of dialog.
    
    ```
    from msquickcmp.pta_acl_cmp.compare import set_task_id
        while True:
            set_task_id()
            query = input("\n用户：")
            if query.strip() == "stop":
                break
            if query.strip() == "clear":
                history = []
    ```
2.  Labeling the pta code
    
    Take patches/models/modeling_chatglm_model.py as an example. This script is used to replace the model granularity.
    
    Compare the precision of the model output with the precision of the model output corresponding to the PTA, find the corresponding code segment, and add the following code:
    
    ```
    from msquickcmp.pta_acl_cmp.compare import set_label, gen_id
    data_id = gen_id()
    set_label("pta", data_id, hidden_states)
    set_label("acl", data_id, acl_model_out[0])
    ```
    
    Compare the SelfAttentionOpsChatglm6bRunner data in the model with the corresponding operator on the PTA side, find the corresponding code segment, and add the following code:
    
    ```
    from msquickcmp.pta_acl_cmp.compare import set_label, gen_id
    data_id = gen_id()
    set_label("pta", data_id, data_val=attention_output)
    set_label("acl", data_id, tensor_path="0_ChatGlm6BModelEncoderTorch/"
    "0_ChatGlm6BLayerEncoderOperationGraphRunner"
                    "/3_SelfAttentionOpsChatglm6bRunner/after/outTensor0.bin")
    ```
3.  Run the comparison command to install the acceleration library, download and compile the acceleration library code, go to the example/chatglm6b directory, and run the following comparison command:
    
    ```
    msit debug compare aclcmp --exec "bash run_performance.sh patches/models/modeling_chatglm_model.py"
    ```
    
     *  **Precautions:**
     *  When performing low-level comparison, you need to perform the inference of the PTA model first and then the inference of the acceleration library.`set_label("acl", data_id, tensor_path)`This operation must be performed before inference on the acceleration library side.
     *  \[\] When low-level comparison is performed, you need to set high-level comparison.
4.  Result analysis
    
    The generated CSV report is as follows:![输入图片说明](%E6%AF%94%E5%AF%B9%E6%8A%A5%E5%91%8A.PNG)	
    
    The following table describes the names of the columns in the CSV file.

| Column name                 | Meaning                                          |
| --------------------------- | ------------------------------------------------ |
| data_id                     | Indicates the data ID.                           |
| pta_data_path               | Path for storing the dump data of the PTA.       |
| pta_dtype                   | Type of PTA data.                                |
| pta_shape                   | Shape of the PTA data.                           |
| pta_max_value               | Maximum value of PTA data.                       |
| pta_min_value               | Minimum value of the PTA data.                   |
| pta_mean_value              | Average value of pta data                        |
| acl_data_path               | Indicates the dump path of ACL data.             |
| acl_dtype                   | Indicates the ACL data type.                     |
| acl_shape                   | Shape of ACL data.                               |
| acl_max_value               | Maximum value of ACL data.                       |
| acl_min_value               | Minimum value of ACL data.                       |
| acl_mean_value              | Indicates the average value of ACL data.         |
| cmp_flag                    | Compare or not                                   |
| cosine_similarity           | Cosine similarity between PTA and ACL data       |
| max_relative_error          | Maximum relative error between PTA and ACL data  |
| mean_relative_error         | Absolute relative error between PTA and ACL data |
| relative_euclidean_distance | Euclidean distance between the PTA and ACL data  |
| cmp_fail_reason             | Cause of the comparison failure                  |

The comparison algorithm is described as follows:

| Comparison Algorithm Name   | Description                                                                                                                                                                                                                       |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| cosine_similarity           | Cosine similarity comparison result. The value range is \[-1, 1\]. If the comparison result is closer to 1, the two values are closer to each other. If the comparison result is closer to --1, the two values are more opposite. |
| max_relative_error          | Indicates the maximum relative error. The value range is \[0, + △). A closer value to 0 indicates a closer value. A larger value indicates a larger difference.                                                                   |
| mean_relative_error         | Indicates the average relative error. The value range is \[0, + △). A closer value to 0 indicates a closer value. A larger value indicates a larger difference.                                                                   |
| relative_euclidean_distance | Euclidean relative distance comparison result. The value range is \[0, + △). A closer value to 0 indicates a closer value. A larger value indicates a larger difference.                                                          |
