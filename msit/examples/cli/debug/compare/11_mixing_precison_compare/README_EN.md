# Precision comparison of hybrid policies #

## Introduction ##

Use the ATC tool to select different floating-point precision modes when converting offline models. To check the precision problems generated by operators in different precision modes, you can compare the precisions of .om models generated in different precision modes.

For details about how to set the floating-point precision of the model in ATC conversion, see.[ATC Tool User Guide - precision_mode_v2](https://www.hiascend.com/document/detail/zh/canncommercial/81RC1/devaids/devtools/atc/atlasatcparam_16_0069.html)	.

## Running an Example ##

Use ATC to convert .onnx models to .om models with different precision policies (Scenario 1 and scenario 2 force_fp16 are used as examples.)

```
atc --framework=5 --soc_version=<soc_version> --model=./resnet50.onnx --output=./resnet50_force_fp32 --precision_mode=force_fp32

atc --framework=5 --soc_version=<soc_version> --model=./resnet50.onnx --output=./resnet50_force_fp16 --precision_mode=force_fp16
```

Run the following command to save the generated .om model in the current directory: (Note: The .om model used after -gm must be the benchmark data of the scenario 1 policy with low convergence degree, that is, force_fp32. The .om model used after -om must be the scenario 2 policy with high convergence degree.) force_fp16:

```
msit debug compare -gm {fp32_om_model_path} -om {fp16_om_model_path} -o {output_file_path}
```

## The result ##

 *  Path to the result_\*.csv file of the Tensor comparison result. For details, see.[Procedure for Comparing Result Analysis](../result_analyse/README.md)	.
 *  The output parameters in the CSV table are different from those in the ONNX model. \[GroundTruth\] and \[NPUDump\] indicate the operator names of the OM offline model in scenario 1 and scenario 2.

| Parameters  | Description                                                                              |
| ----------- | ---------------------------------------------------------------------------------------- |
| GroundTruth | Indicates the operator name of the offline model for the precision policy in scenario 1. |
| NPUDump     | Indicates the operator name of the offline model in the precision policy in scenario 2.  |
